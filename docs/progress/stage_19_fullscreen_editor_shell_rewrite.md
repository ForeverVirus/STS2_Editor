# Stage 19 - Fullscreen Editor Shell Rewrite

## 已开发内容
- `Mod Studio` 的项目编辑界面已从旧的 `home/workspace + drawer` 混合结构，切换为新的全屏三栏壳层。
- 顶部改成窄菜单栏：
  - 模式切换
  - 项目选择器
  - 新建/复制/删除项目
  - 新建条目
  - 导出安装
  - 语言切换
  - 右上角退出按钮
- 项目模式不再依赖旧的项目首页才能进入编辑区，打开 `Mod Studio` 后直接进入编辑工作区。
- 主编辑区改成固定三栏比例：
  - 左侧约 20%：实体类型 Tab + 搜索 + 范围筛选 + 列表
  - 中间约 60%：`基础信息 / 资源 / Graph`
  - 右侧约 20%：跟随中间页签自动切换的详情面板
- 左侧实体浏览器已改为真正的分页签结构，不再用原来的单排按钮分类。
- 中间资源页改成双预览对比：
  - 当前资源
  - 候选预览
- 右侧资源详情区改成三段式：
  - 当前绑定
  - 游戏内素材
  - 项目已导入素材
- Graph 区保留 `GraphEdit + GraphNode` 画布，并新增右侧节点检视器：
  - 节点类型
  - 节点 ID
  - 节点显示名
  - 节点描述
  - 节点属性列表
- 新增右上角退出流程：
  - 有未保存修改时弹窗提示
  - 支持保存后退出 / 丢弃退出 / 取消
- `NSubmenu` 自带的左下角红色返回按钮现在会在 `_Ready`、`OnSubmenuOpened`、`OnSubmenuShown` 阶段主动隐藏，避免遮挡编辑器。
- 双语文案补到了这次新 UI 需要的菜单、退出弹窗、资源对比、节点检视器等区域。

## 未开发内容
- 这轮重写主要是壳层与交互结构重做，还没有做新的美术风格统一设计。
- 资源页目前仍然以贴图对比预览为主，还没有补成“真实卡牌节点 vs 候选卡牌节点”的并排卡牌预览。
- Graph 目前已经是可拖拽画布，但还不是完整的 UE 蓝图式体验：
  - 还没有右键菜单加点
  - 还没有缩放辅助、小地图、框选工具
  - 还没有更强的节点面板拖拽创建
- 包管理页这轮没有做同等级的全面视觉重构，重点仍然放在项目编辑页。
- 还没有补“切换项目时如果存在未保存修改也弹确认”的保护。

## 遇到的问题
- 旧 `NModStudioScreen` 里 UI 和业务逻辑耦合很深，直接删掉所有旧方法风险太高，所以这轮采用的是“新增重写壳层 + 保留原有后端逻辑接线”的方式。
- `NSubmenu` 基类会在显示时重新启用原版返回按钮，所以不能只在一个生命周期里隐藏，必须多次调用 `HideBackButtonImmediately()`。
- Graph 右侧节点检视器在刷新时会触发文本框信号，需要额外加一层 `_suppressGraphInspectorChanged`，否则会出现自触发写回。
- Windows 下 `apply_patch` 超长补丁会触发命令长度限制，所以这轮 UI 重写分块落地到了新的 partial 文件中。

## 后续如何解决
- 先进入真实游戏做一轮 UI 实测，重点确认：
  - 是否真正全屏占满
  - 左侧分页签列表是否足够高、足够可读
  - 右侧资源面板是否终于可用
  - Graph 画布是否已经回到中心主舞台
- 如果实测壳层结构没问题，下一轮继续做两件事：
  - 资源页补成“真实卡牌/遗物/药水外观对比预览”
  - Graph 继续向蓝图式交互推进
- 在项目切换、模式切换上继续补未保存保护，避免误丢编辑内容。
