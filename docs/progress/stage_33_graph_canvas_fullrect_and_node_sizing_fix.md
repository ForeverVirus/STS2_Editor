# Stage 33 - Graph Canvas FullRect And Node Sizing Fix

## 已开发内容
- 根据真实游戏截图，修复了 `Graph` 画布宿主尺寸链：
  - `ModStudioGraphCanvasView` 顶层控件强制 `FullRect`
  - 内部根控件与 `GraphEdit` 容器改为真正铺满画布区域
- 修复节点显示过窄的问题：
  - 为 `GraphNode` 设置固定初始宽度和最小高度
  - 节点标题/类型/属性摘要标签改为默认不单字换行
  - 端口行改为固定宽度布局，避免看起来像一列竖排文本
- 打开 graph 时自动执行一次 `Zoom To Fit`，避免旧布局或偏移导致节点落在视野外。

## 未开发内容
- 还没有把 graph 做到完整 UE 蓝图那种多选、框选、小地图、拖线到空白处自动生节点的成熟交互。
- 还没有把数据 pin 的类型约束和运行时执行完全打通，目前仍是以流程图 + 节点属性驱动为主。

## 遇到的问题
- 最新用户截图表明，上一轮虽然已经切成了画布模式，但内部宿主尺寸没有真正铺开，导致 `GraphEdit` 可用区域和节点尺寸异常，表现为画布像一条窄列表。
- 节点标签默认自动换行，配合异常的节点宽度后，会直接退化成竖排单字文本。

## 后续如何解决
- 如果这轮实机验证后画布已恢复正常，下一轮继续收 Graph 体验：
  - 右键菜单分类
  - 拖线到空白处创建节点
  - 端口类型约束
  - 小地图和快捷键
- 如果仍有尺寸异常，需要继续沿实际游戏截图查 `GraphEdit` 在宿主层级中的最终 `Rect/Size`。
