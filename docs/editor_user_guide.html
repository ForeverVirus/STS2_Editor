1. 编辑器是什么

STS2 Editor 是一个面向 Slay the Spire 2 模组内容的可视化编辑器。它把项目文件、素材绑定、Graph 行为图、包导出与已发布包管理整合到同一套界面里。

你可以把它理解成“工程编辑器 + 包管理器”两部分：前者负责制作内容，后者负责安装、刷新和排查发布后的包。
对象类型 可编辑内容
角色 基础字段编辑。角色当前没有 Graph 标签页。
卡牌 / 遗物 / 药水 基础字段、素材绑定、Graph 行为。
事件 基础字段、Graph 行为，适合做多页事件、奖励、战斗恢复页等。
附魔 基础字段、Graph 行为，可配合状态与钩子逻辑。

2. 两种模式

Project Mode

用于创建和编辑工程。你会在这里做对象覆盖、新建条目、素材导入、Graph 编排、导出包。

Package Mode

用于管理已经导出的 .sts2pack 模组包，查看安装状态、冲突信息并执行热重载。

从模式选择界面进入后，Project Mode 负责“做内容”，Package Mode 负责“管成品”。

3. Project Mode 工作流

  1. 进入 Project Mode 后，先点击顶部菜单里的“新建项目”或“打开项目”。未打开项目时不会进入完整编辑界面。
  2. 在左侧实体浏览器选择要编辑的对象类型，例如卡牌、遗物、事件。
  3. 点击某个条目进入编辑；如果要做新内容，先切到目标类型后点击“新建条目”。
  4. 在中间主编辑区修改基础字段、素材或 Graph；在右侧面板查看附加信息、资产列表与 Graph 细节。
  5. 保存当前修改后，通过“导出包”生成 .sts2pack，再去 Package Mode 热重载验证。
切换对象或退出时，如果当前有未保存内容，编辑器会弹出保存确认。建议形成“改一段就保存一次”的习惯。

4. 基础信息页

中间区域的“基础信息 / Basic”标签页用于编辑当前对象的元数据字段。这里会根据条目类型动态生成字段控件。

  • 普通文本会显示为单行输入框或多行文本框。
  • 布尔值会显示为勾选框。
  • 带候选项的字段会显示为下拉选择器。
  • 列表字段会显示为可增删的条目列表。

保存

点击“保存”会把当前页字段写回工程覆盖数据。

还原

点击“还原”会丢弃当前基础字段页尚未提交的编辑。

5. 资源页

支持资源的对象会显示“资源 / Assets”标签页。这个页面用于把运行时素材或项目外部素材绑定到当前条目。

区域 作用
右侧“游戏内素材” 从当前游戏已有素材中选择一个资源路径作为绑定目标。
右侧“导入的外部素材” 管理项目导入的 PNG/JPG/JPEG/WEBP 文件。第一项可直接新增外部素材。
中间预览区 对比当前预览与候选预览，确认后点击“保存应用”。
外部素材导入后会被复制到当前项目目录下的 assets 结构中，而不是只记录原始绝对路径。

6. Graph 页

Graph 页面负责编辑行为逻辑。卡牌、遗物、药水、事件、附魔支持 Graph 标签页；角色没有这一页。

顶部工具栏

  • 导入：从模板或候选图开始。
  • 添加节点：在画布中心打开节点选择。
  • 删除节点：删除当前选中的节点。
  • 自动布局 / 缩放适配：整理画布和视角。
  • 保存 Graph / 校验：保存当前图并做结构检查。

右侧 Graph 详情

  • 启用 Graph 行为：切换当前条目是否使用 Graph 覆盖原生行为。
  • Graph ID / Graph 名称 / Graph 描述：图级元数据。
  • 节点属性编辑：选中节点后右侧会显示属性、上下文帮助和部分事件专用选择器。
只改 Graph 描述文字不会自动改变实际奖励或行为。尤其是事件奖励，真正生效的是对应节点和属性值。

事件 Graph 额外支持页面 ID、选项 ID、返回页、奖励类型、奖励目标等上下文字段。右侧面板会针对这些属性给出更贴近业务的解释。

7. Package Mode

Package Mode 用来查看已经发布的包，并把最新导出的包快速同步到游戏运行时。

.sts2pack 指定目录：编辑器默认扫描的已发布包目录是 游戏目录\mods\STS2_Editor\mods\。 例如常见路径会是 F:\SteamLibrary\steamapps\common\Slay the Spire 2\mods\STS2_Editor\mods\。 如果你把 .sts2pack 放在别的目录，Package Mode 默认不会显示它。
  1. 进入 Package Mode 后，左侧会列出上面这个指定目录里的已发布包，可以按名称、作者、描述搜索。
  2. 点中某个包后，右侧会显示基本信息和冲突详情。
  3. 顶部点击“Hot Reload”可刷新已发布包目录，让当前游戏运行时重新读取包内容。
推荐做法是：Project Mode 导出包时直接保存到这个指定目录，然后立刻切到 Package Mode 执行一次 Hot Reload，再进入游戏内验证。

9. 常见问题

为什么有的对象没有资源页或 Graph 页？

编辑器会按对象类型动态显示功能。角色没有 Graph 页；只有支持资源绑定的对象才会显示资源页。

为什么我改了文字，但实际效果没变？

基础字段、Graph 描述和实际行为是两层概念。真正的逻辑变化通常要落在 Graph 节点、属性值、奖励类型或绑定资源上。

为什么点击“使用说明”或“预制键说明”没有打开？

通常是说明文件还没有被编译/同步到当前模组目录。重新构建当前项目后再试一次。

如何快速确认内容有没有写进项目？

看顶部项目状态是否显示“已保存”，再导出一遍包并去 Package Mode 热重载。

10. 推荐发布流程

  1. 在 Project Mode 打开项目并完成对象编辑。
  2. 逐页保存：基础字段、素材绑定、Graph。
  3. 用 Graph 页里的“校验”先做一次结构检查。
  4. 通过“文件 → 导出包”输出新的 .sts2pack，并保存到 游戏目录\mods\STS2_Editor\mods\。
  5. 切到 Package Mode，确认包列表能看到新版本或新时间戳。
  6. 点击“Hot Reload”。
  7. 进入游戏实际验证表现，必要时返回继续调试。
